**提案者**: 鳥居万椰・村上貴人・小関萩花（筑波大学大学院 情報学・看護学 博士後期課程）

## ① プロジェクトの背景、目的、目標

### 背景：二重の孤立が解決されていない

日本の独居高齢者は約750万人（2020年国勢調査）。そのうち軽度認知障害（MCI）を持つ人は約400万人と推計される。この層にとって、日常の「誰かに話しかける」行為はほぼ失われており、孤独感・認知機能低下・情報断絶が三位一体で進行する。

問題の核心は「話し相手がいない」ことだが、それだけではない。高齢者の孤立は同時に**家族のつながり断絶**でもある。スマートフォンを使える親であればLINEで日々やり取りして安否を確認できる。そうでない親を持つ家族は「今日どうだったか」を知る術がなく、多忙な日常の合間を縫って物理的に様子を見に行くしかない。

家族が見守りたいのは認知機能の低下のような大きな変化だけではない。「トイレで立ち上がれず動けなくなっていないか」「薬を飲み忘れていないか」「猛暑日に室温が危険でも気づかずに過ごしていないか」——高齢者の生活は、こうした小さな日常のほころびから揺らいでいく。電話は相手が出なければ終わり、スマホは親が使えない。こうした生活の揺らぎを遠くから見守れるプロダクトは、現在存在しない。

「Alexaがあるのでは」という問いへの答えも明確だ。スマートスピーカーはウェイクワードを思い出し、声量を出して発話できる人を前提とする。75歳以上での偽陰性率は若年層の2〜5倍に達し（Vipperla et al., 2010）、記憶も関係性も身体もない。毎回ゼロから話しかける「道具」は、孤独を解かない。

```
【図1：既存ソリューションが問題を解かない理由】

  独居高齢者の二重の孤立        既存デバイスの限界
  ┌──────────────────┐   ┌──────────────────────────────────┐
  │ 話し相手がいない    │──▶│ スマートスピーカー：ウェイクワード失敗多数 │
  │ 家族とつながれない  │──▶│ スマートフォン：操作自体が障壁         │
  │ 生活の揺らぎが見えない│──▶│ 見守りセンサー：会話・関係形成機能なし  │
  └──────────────────┘   └──────────────────────────────────┘
              ↓
    「操作しなくていい、そこにいてくれる存在」が必要
```

### 目的：「そこにいる存在」としての身体化AIエージェント

**Suzume-chanは、独居高齢者の部屋に置くと自然に「一緒にいる」ぬいぐるみ型AIエージェントである。** 子や孫世代が購入して実家に送る——高齢者本人がセットアップする必要はなく、ただそこに置かれるだけでよい。

声をかければ応じる。毎日少しずつ好み・思い出・話し方を覚えていく。ペットや古い友人のように**関係が育つ**。この「いる感（Social Presence）」と触れられる身体性こそが、スマートスピーカーやセンサーとの根本的な差異である。

家族のスマートフォンには毎晩「今日のサマリー」が届く。さらに、「薬を飲んだか確認してほしい」など家族からの依頼をSuzume-chanが会話の中で自然に聞き、結果を家族のアプリに通知する。高齢者と家族を結ぶ**双方向の情報連携**が核心機能である。

## ② 開発に関する未踏性の主張、期待される効果

### 未踏性の核：「今服用している薬は覚えているが、1年前の朝ごはんは覚えない」3層記憶アーキテクチャ

本プロジェクトの技術的中核は、**ヒトの記憶構造を模した3層オンライン適応システム**である。既存の会話AIはRAGによる検索か静的プロンプト注入に留まり、「何を長く覚えるべきか・何を忘れるべきか」を自動判断する機構を持たない。本提案は記憶の重要度・時間減衰・再参照頻度から適切な記憶層への**自動ルーティング**を実装する。

この3層設計は記憶の精度だけでなく、**応答速度の保証**にも直結する。高齢者との会話では長い沈黙が「壊れた」と知覚されるため、発話終了から最初の音声出力まで2.5秒以内が要件となる。適切な記憶層のみを高速参照することで不要なRAG検索を省き、LLMへの入力トークン数を制御して応答速度目標を実現する。

```
【図2：3層記憶アーキテクチャとルーティング】

  会話入力 → ルーティングエンジン（MIS = 参照頻度×感情強度×時間近接性×カテゴリ重み）
                  │                    │                      │
           ▼ 短期記憶              ▼ 中期記憶             ▼ 長期記憶
      ┌──────────┐        ┌──────────────┐     ┌────────────────┐
      │Context   │        │ RAG (FAISS)  │     │ LoRA Adapter   │
      │Window    │        │ ~1,000エントリ │     │ 週次バッチ Rank-8│
      │~10ターン  │        │ 数日〜数ヶ月   │     │ 数ヶ月〜永続     │
      └──────────┘        └──────────────┘     └────────────────┘
  「さっき言った薬」     「先週娘が来た」         「ゆっくり話す」
  「今何の話？」        「病院の予約3/25」        「鳥の話題が好き」
                       「最近眠れていない」       「敬語より友達口調」

  例：1年前の朝ごはん → MIS低→短期記憶のみ（セッション後破棄）
     今服用している薬 → MIS高→中期/長期記憶に保持・継続参照
```

**なぜ3層か：**

| 手法 | 限界 |
|---|---|
| 静的プロンプトのみ | セッションを跨ぐ記憶がなく個人適応が浅い |
| RAGのみ | 過去ログが等価に検索され「何を重視するか」を学習しない |
| オンラインLoRAのみ | VRAM2〜4倍・勾配不安定、実運用環境では非現実的 |
| **本提案：3層自動ルーティング** | 重要度に応じた記憶層選択により計算コスト・安定性・適応深度を最適化 |

### 未踏軸①：認知負荷ゼロのウェイクワード不要・身体化トリガー起動

```
【図3：3ステージ起動アーキテクチャ】

  Stage 1: 近接ゲート（~10mW 常時待機）
    BLE RSSI + PIRモーション → ユーザ存在検知。人がいない間はマイクを起動しない

  Stage 2: 物理トリガー（~25mW）
    IMU（持ち上げ検知）＋ 導電布タッチ（握り検知）→ 応答モード起動（LED点灯）
    「持ち上げれば話せる」= 作業記憶を使わない起動

  Stage 3: 音声活動検知（~45mW）
    Silero VAD on RP2040 → 発話終端検知 → Opus圧縮 → WiFi送信
```

ウェイクワード方式では75歳以上の偽陰性率が若年層の2〜5倍に達する（Vipperla et al., 2010）。「Hey Alexa」を思い出して声量を出すという行為は、MCIで最初に崩れる作業記憶を要求する。本提案は「持ち上げる・握る」という運動記憶ベースの操作に置き換えることで、この障壁を根本から解消する。

LLM推論中（1〜2秒）は振動触覚フィードバックにより「聴いているよ」「考えているよ」を身体的に伝達する。我々の予備実験（HRI '26 ワークショップ、27名定性評価）では、振動なし条件で推論中の沈黙が「壊れた」と知覚されて繰り返し発話が多発したのに対し、振動あり条件では自然な待機が観察された。この「待てる感覚」が継続使用の鍵となる。

### 未踏軸②：API優先アーキテクチャと将来ローカル移行の二段設計

```
【図4：システム全体アーキテクチャ】

  [ぬいぐるみデバイス] RP2040 + RPi Zero 2W（~30mW待機）
   3ステージ起動 / Opus音声ストリーミング / 振動触覚制御(DRV2605L)
        │ WiFi（家庭LAN / 施設LAN）
        ▼
  [バックエンドサーバ（抽象化APIレイヤー）]
   ASR: distil-Whisper / Whisper API
   LLM: OpenAI API / Claude API / OSS APIサーバー（切替可能）
   3層記憶ルーティングエンジン ← 本提案の技術核
   RAG: FAISS / TTS: Style-Bert-VITS2 / 家族向けサマリー生成

  ┌──────────────────────┬────────────────────────────┐
  │ プロダクト開発ルート    │ 研究ルート（並行）           │
  │ API優先・高速開発      │ 完全ローカル（施設・医療向け）  │
  │ スケール容易           │ ローカルLLM・プライバシー規制対応│
  └──────────────────────┴────────────────────────────┘
  目標RTT：発話終了→最初の音声出力 2.0〜2.5秒（API利用時）
```

在宅向けプロダクトをAPI優先で高速開発し、施設展開フェーズでオンプレミスへ段階移行する。研究サイドでのローカルLLM実証が移行の技術的根拠となり、クラウド依存の大企業が追随しにくい護城河として機能する。

### 克服すべき課題と解決策

| リスク | 優先度 | 解決策 |
|---|---|---|
| 3層記憶ルーティングの精度 | 高 | 初期はMISパラメータを手動調整→実証データでチューニング。LoRA（長期記憶）層がなくても短期・中期記憶だけで主要価値は成立 |
| LoRA学習データの倫理同意設計 | 高 | LoRAは同意取得参加者に限定。同意を得られない場合は短期・中期記憶層のみで機能完結 |
| RP2040上のSilero VAD実行可能性 | 中 | 代替：VAD処理をZero 2Wに移管（有線給電前提で常時電力~300mW） |
| 在宅高齢者リクルート | 中 | 小関の看護師ネットワーク・筑波大学地域連携部門経由。まず10名から開始 |

## ⑤ 事業化・社会実装の新規性・優位性、想定するターゲットと規模

### ターゲットと購買フロー

**購買者（一次ターゲット）**：離れて暮らす子・孫世代（30〜50代）。父の日・敬老の日・誕生日ギフトとしてECで購入し実家に送る。初期設定はスマートフォンアプリで遠隔完結——高齢者本人はただ置かれたものと話すだけでよい。

**エンドユーザ**：軽度認知障害または軽度身体障害を持つ独居高齢者（65〜85歳、要介護0〜1相当）。

**将来展開**：在宅での実績・ブランド認知を積んだ後、高齢者施設への展開を検討。本事業期間のスコープは在宅（CtoC）に集中する。

### 競合比較

| 競合 | 限界 |
|---|---|
| Alexa / Google Home | ウェイクワード必須・記憶なし・医療規制非対応 |
| パロ（AIST） | 感情接触特化・受動デバイス。情報・ログ機能なし |
| ペッパー（SoftBank） | 月額¥100万超・UIあり・家庭利用不可 |
| 見守りセンサー | 状態検知のみ。会話・関係形成・記憶機能なし |
| **Suzume-chan** | ウェイクワード不要・3層記憶・身体性・家族連携・ローカル移行可 |

### 新規性の核：スマホ非利用者の生活ログデータ基盤

スマホを使えない高齢者（約1,000万人規模）の日常データは現在どこにも存在しない。Suzume-chanの普及とともに蓄積される会話・感情・生活ログは、製薬・保険・医療研究機関にとって高い価値を持ち、opt-in同意を前提とした匿名化データ提供が将来の収益柱となる。
