**提案者**: 鳥居万椰・村上貴人・小関萩花（筑波大学大学院 情報学・看護学 博士後期課程）

## ① プロジェクトの背景、目的、目標

### 背景：操作を前提とするUI設計が生む構造的排除

日本の65歳以上人口（3,640万人）のうち、認知症患者（約700万人）・入院患者（約130万人/日）・重度運動障害者（約200万人）は、既存のデジタル情報端末を実質的に使用できない。問題の本質は「デジタルリテラシーの欠如」ではなく、**操作を前提とするUI設計のパラダイムそのもの**にある。スマートスピーカー（Alexa/Google Home）ですらウェイクワード発話が必要であり、75歳以上では偽陰性率が若年層の2〜5倍に達する（Vipperla et al., 2010）。

```
【図1：なぜ既存ソリューションが失敗するか】

  利用困難の原因          失敗の構造
  ┌──────────────┐     ┌─────────────────────────────────┐
  │ウェイクワード依存 │────▶│ Siri/Alexa/Google：同じ問題を持つ │
  │画面操作の困難    │────▶│ タブレット支援：操作自体が障壁   │
  │情報提供のタイミング不一致│▶│ ICT看護：プッシュ型のみ         │
  │記録の非対称性    │────▶│ 電子カルテ：記録コストが増大     │
  └──────────────┘     └─────────────────────────────────┘
         ↓
    「操作できる人」を前提とする設計を変えるしかない
```

厚生労働省の推計では2030年に介護職員不足は約69万人。情報断絶は服薬誤り・治療拒否・孤立による認知症進行加速として連鎖的社会コストを生む。日本は世界最速で超高齢化社会に突入しており、ここで開発・実証されたソリューションは韓国・ドイツ・シンガポールが2030〜40年代に直面する課題の先行解として輸出可能なカテゴリになる。

### 目的：「身体化AIターミナル」によるパラダイム変換

**Suzume-chan**は、ぬいぐるみ型物理デバイスと施設内サーバ上の音声対話・適応・ログ生成システムを統合した**身体化AIターミナル（Embodied AI Terminal）**である。「Alexa in a plush toy」ではない。4点で根本的に異なる：

| 差異 | Alexa / スマートスピーカー | Suzume-chan |
|------|--------------------------|-------------|
| 起動 | ウェイクワード必須 | 持ち上げる・話しかけるだけ（IMU+VAD複合トリガー）|
| 記憶 | セッション内のみ | 4層適応で長期関係性を形成 |
| 身体フィードバック | なし | 振動触覚で「聴いている」「考えている」を伝達 |
| 双方向性 | 単方向（ユーザ→クラウド）| 患者→情報取得 + 会話→看護ログ生成 |

### 目標（事業期間：2026年7月〜2027年2月、8ヶ月）

**技術目標**：エンドツーエンド応答遅延2.5秒以内 / IMU+VADによるウェイクワードゼロ起動 / 4層適応システムv1実装 / デバイスv2プロトタイプ（BOM ¥10,000以下）完成

**社会実装目標**：高齢者施設2施設以上での実証実験（各2週間以上）/ 医療倫理審査承認取得 / 量産プロトタイプv2の製造パートナーとの原価合意

**競合との差別化**：Amazon/Googleはクラウド依存でありプライバシー規制上医療・介護環境への展開が困難。パロ（AIST）は感情接触特化の受動デバイスであり情報・記憶・ログ機能を持たない。完全オンプレミス設計と低エージェンシーユーザ向け臨床設計により**規制が護城河として機能する**。

## ② 開発に関する未踏性の主張、期待される効果

### 未踏軸1：ウェイクワード不要・3ステージ複合トリガー起動アーキテクチャ

既存の全ての音声インターフェースは能動的起動行為を前提とする。本提案は**認知負荷ゼロの複合トリガーアーキテクチャ**を設計・実装する。

```
【図2：3ステージ起動アーキテクチャ】

  Stage 1: 近接/存在ゲート (~10 mW)
    BLE RSSI + PIRモーションセンサ → ユーザ存在検知
    ※このゲートが開いていない間はStage 2,3は起動しない

  Stage 2: 物理インタラクショントリガー (~25 mW)
    IMU (LIS2DH12): ピックアップ検知 ──┐
    静電容量タッチ（導電布）: 握り込み ──┤→ 音声受付モード起動
                                        │
  Stage 3: 音声活動検知 (~45 mW)      ←┘
    Silero VAD (RP2040 TFLite-Micro: ~3 ms/chunk)
    2chマイクアレイ + ビームフォーミング
    → Opus圧縮 → WiFi送信

  【比較】ウェイクワード偽陰性率 75歳以上 2〜5倍 → 物理トリガーで解消
```

### 未踏軸2：破滅的忘却を防ぐ4層適応システム

単純なRAGではなく、安全性を担保しながら段階的に深める4層アーキテクチャ：

| 層 | 内容 | 更新頻度 |
|----|------|---------|
| L0: 固定ベースLLM | Llama 3.2-14B Q4_K_M（~8GB）基盤、更新しない | 固定 |
| L1: プロファイル注入 | 名前・服薬・話題傾向をシステムプロンプト注入（800トークン）| 毎セッション |
| L2: エピソード記憶RAG | 会話サマリをFAISSインデックス化、トップ5を自動注入（~30ms）| セッション間累積 |
| L3: 行動ポリシーモジュール | 発話速度・ポーズ長に応じTTS速度・文長を動的制御 | セッション内 |
| L4: LoRAバッチ適応 | 週次バッチ学習（倫理審査承認後、研究フェーズ） | 週次 |

破滅的忘却対策：学習データに汎用日本語対話サンプル10〜20%を混入（リプレイバッファ）。安全フィルタによる品質劣化検出時は直前バージョンにロールバック。

### 未踏軸3：振動触覚による待機時間の相互作用化

HRI '26実証（27名）の主要知見：**LLM推論中1〜2秒の沈黙は「無反応」として知覚され、繰り返し質問が発生する**。振動触覚があると同じ遅延が「考えている」として自然受容される。これはUXの話ではなく、繰り返し質問によるシステム負荷・コンテキスト汚染を防ぐ機能的問題解決である。

3段階振動パターン（DRV2605L）：Listening（0.5Hz穏やかなパルス）→ Thinking（2Hz連続振動）→ Speaking（3×短バースト予告）

### 未踏軸4：施設展開可能な分散推論アーキテクチャ

```
【図3：システムアーキテクチャ】

  [ぬいぐるみデバイス] RP2040 + RPi Zero 2W
   ● 3ステージ起動ロジック       常時待機: ~30 mW
   ● Opus音声ストリーミング送信
   ● 振動触覚制御 (DRV2605L)
          │ WiFi 802.11n（施設LAN内, RTT<15ms）
          ▼
  [施設サーバ] Mac mini M4 Pro 48GB
   ● distil-Whisper ASR:  200–400 ms
   ● 14B Q4 LLM (llama.cpp Metal): TTFT ~1,000–2,000 ms
   ● FAISS RAGシステム:      ~30 ms
   ● Style-Bert-VITS2 TTS:  400–700 ms（ストリーミング）
   ● 看護ダッシュボード + 週次LoRAバッチ学習ジョブ

  目標RTT（発話終了→最初の音声出力）: 2.0–2.5秒
  同時接続: 最大8台（時分割処理）
```

**期待される効果**：定型問い合わせの代替による介護士の本来業務への集中、会話からの看護記録自動ドラフト生成による記録業務削減、入居者の孤立感・情報アクセス格差の解消。

## ⑤ 事業化・社会実装の新規性・優位性、想定するターゲットと規模

### カテゴリ定義：「身体化AIターミナル」は既存市場に存在しない

「ウェイクワード不要 × ローカル処理 × 長期記憶 × 双方向看護ログ × 身体化」を同時に実現した商用製品は現時点で存在しない。Amazon/Googleはクラウド依存で医療情報規制上の展開が困難、ソフトバンクRobotics（Pepper）は月額¥100万超で施設経済性に合わない、パロ（AIST）は受動デバイスで情報・記憶機能を持たない。完全オンプレミス設計が規制護城河として機能し、大企業が追随できない差別化要因となる。
