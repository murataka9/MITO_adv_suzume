# Suzume-chan：投資家・事業化メモ

**作成目的**: MITOU提案書（academic向け）を補完する、投資家・パートナー向けの事業化ロジックの整理

---

## 1. プロダクト概要

**Suzume-chanは「低エージェンシーユーザ向け身体化AIターミナル」である。**

スマートフォンを使えない高齢者・認知症患者・入院患者に対して、ウェイクワード不要・操作不要の音声インターフェースを提供する。ぬいぐるみ型物理デバイス（RP2040+RPi Zero 2W、BOM ¥8,850）と施設内サーバ（Mac mini M4 Pro）で構成される完全オンプレミス型。

**3秒で使い方を説明できる**: 「持ち上げて話しかけるだけ。操作もウェイクワードも不要」

---

## 2. 問題と機会

### 問題
日本の65歳以上人口3,640万人のうち、認知症患者約700万人・入院患者約130万人/日は既存のすべてのデジタル情報端末を使えない。スマートスピーカーですら「Hey Alexa」というコマンド記憶と発話が必要であり、認知機能低下・声量低下のユーザには機能しない（偽陰性率2〜5倍）。

### なぜ今か
- 日本の高齢化率2025年に29.1%。介護職員不足が2030年に69万人に達する
- 施設IoT導入の意欲は高まっているが、「高齢者が実際に使える」プロダクトが存在しない
- LLMの推論コスト低下により施設内オンプレミスが経済的に実現可能になった（Mac mini M4 Pro 14B Q4推論で月電気代¥1,500相当）

## 4. 顧客定義（ICP）

**初期理想顧客プロファイル（Ideal Customer Profile）**:

- 入居者50〜200名規模の介護老人福祉施設（特養）または有料老人ホーム
- 施設長または事務長が「介護DX」に関心を持っている
- 職員の離職率・負担に課題感がある
- 茨城・関東圏（初期）

**典型的な購買プロセス**:
1. 施設長が業界展示会またはネットワーク紹介でSuzume-chanを知る
2. 「お試し3ヶ月プラン」（5〜10台レンタル、月¥5,000/台）でトライアル開始
3. 3ヶ月後に稼働データ（問い合わせ削減率・職員評価）を確認してフルロールアウト意思決定
4. 全室展開（20〜60台）＋月額サービス契約

## 5. 価格設計

| 製品/サービス | 価格 | 粗利率 |
|---|---|---|
| デバイス（量産品） | ¥35,000/台 | 49%（¥17,000/台） |
| 月額サービス | ¥2,500/台/月 | 90%以上（原価はインフラ・サポート人件費） |
| 初期導入支援 | ¥100,000〜¥300,000/施設 | 60%（人件費依存） |

**ユニットエコノミクス（30台契約施設）**:
- 初期収益：30台 × ¥35,000 + ¥200,000（導入支援）= ¥1,250,000
- 月次収益：30台 × ¥2,500 = ¥75,000/月
- LTV（3年継続仮定）：¥1,250,000 + ¥75,000 × 36 = **¥3,950,000**
- CAC（展示会・直販）：推定¥150,000〜¥300,000/施設
- **LTV:CAC ≒ 13〜26x**（サブスク継続率85%仮定）

## 6. ゴートゥーマーケット

**ウェッジ（楔）**: 茨城県内の介護施設2施設（Agri Care社・筑波大学関連）での実証実験が進行中。この2施設でのケーススタディ＋論文が最初の販売ツールになる。

**拡大経路**:
1. **垂直深化（医療法人ネットワーク）**: Agri Careでの成功 → 同医療法人グループ内の他施設へ横展開。1法人に入ると複数施設への展開が加速
2. **水平展開（学術ネットワーク）**: 筑波大学・立命館大学経由の施設紹介。学術論文をトリガーにした施設からのインバウンド問い合わせ
3. **展示会チャネル**: CARETEX（東京・大阪）・Care Show Japan。施設長が集まる場への年1〜2回出展
4. **OEM/チャネルパートナー**: 大手介護事業者（ニチイケアセンター・ベネッセスタイルケア）との一括採用商談（FY2028〜）

## 8. 競合優位性のまとめ

| 軸 | Suzume-chan | Amazon Echo / Google Home | PARO | 介護記録SaaS |
|---|---|---|---|---|
| ウェイクワード不要 | ✓ | ✗ | ✗（対話なし） | ✗ |
| 完全ローカル処理 | ✓ | ✗ | ✓ | 一部 |
| 長期記憶・個人適応 | ✓ | ✗ | ✗ | ✗ |
| 看護ログ生成 | ✓ | ✗ | ✗ | ✓ |
| 身体化（物理的存在感） | ✓ | ✗ | ✓（感情のみ） | ✗ |
| BOM¥10,000以下 | ✓ | — | ✗（¥40万） | — |

**護城河（Moat）の3層**:
1. **規制Moat**: クラウド処理が事実上禁止される医療環境でAmazon/Googleは参入困難
2. **臨床パートナーシップMoat**: 施設実績・看護師ワークフロー統合・倫理審査通過の実績は短期複製不可
3. **データMoat**: 入居者プロファイル・施設固有知識DBの蓄積がスイッチングコストを形成

## 9. キーリスク

| リスク | 対応 |
|---|---|
| 施設の意思決定が遅い（3〜6ヶ月） | 2施設の実証を2026年内に完了し、2027年の営業開始時点で「データあり」状態にする |
| デバイス製造コストが下がらない | 少量（50〜100台）での実績を作り、500台ロットでBOMを¥15,000以下に圧縮 |
| 大企業が「十分良いもの」を出す | 医療環境向け完全オンプレミス、論文実績、施設特化機能は短期では複製困難 |
| 規制分類が予想外（医療機器扱い） | 2026年9月にPMDA相談完了、「生活支援機器」として設計済み（診断・治療に関与しない） |
| LoRAバッチ学習の倫理承認が遅延 | Layer 1〜3だけでも製品価値は成立。LoRAはオプショナル |

## 10. チームのなぜ

3名の創業チームがこの問題を解く最適な人材である理由：
- **鳥居万椰**: プロトタイプv1のハードウェア設計・統合を主導。HCI評価設計の専門家
- **村上貴人**: 現プロトタイプのAIパイプライン全段を実装。LLM/RAG/TTS/ASRの実装実績あり
- **小関萩花**: 看護学専門家かつ施設ネットワーク保有。倫理審査・規制対応を担当できる唯一のメンバー

この3つのスキルセット（HCI×AI×医療連携）の同時保有は他のチームには容易に複製できない。さらに、HRI '26 ACM/IEEEへの採録という国際的な学術的信頼性が裏打ちする。
