Suzume-chan プロジェクト コンテクスト（未踏アドバンスド用）

0. プロジェクトの核（最重要）

本プロジェクトの本質は、

「スマホを使えない人のための、身体化されたスマートフォン」

である。

既存のスマートフォンやAIアシスタントは「操作できる人」を前提としている。一方で、高齢者・患者・認知機能が低下したユーザ・デジタルリテラシーの低いユーザにとって、
UIそのものが障壁になっている。

Suzume-chanはこの問題を、
	•	デバイスを操作させない
	•	音声＋身体性＋関係性で代替する

ことで解決する。

⸻

1. 既存研究と技術シーズ（2本の核論文）

(1) Embodied Information Hub
	•	身体を持つAIが知識伝達を媒介する概念
	•	Social Presence（「そこにいる感」）を活用

👉 Suzume-chanは
「情報アクセス」ではなく「関係性の中での理解」を提供する

￼
	•	LLM + RAG + 音声 + 物理エージェント
	•	個人DBを学習し、継続的関係を形成
	•	「知識の非同期共有」を可能にする

⸻

(2) 医療・看護応用（HRI）
	•	双方向メディエーション
	•	患者 → 情報取得
	•	看護師 → 状態ログ取得

￼
	•	情報提供の「タイミング不一致」を解決
	•	会話ログが看護データになる
	•	ローカルLLMでプライバシー確保

👉 重要ポイント：

「単なる会話AIではなく、情報インフラとして機能する身体エージェント」

⸻

2. 解くべき社会課題

問題構造（抽象化）
	1.	情報はあるがアクセスできない
	2.	UIが複雑すぎる
	3.	タイミングが合わない
	4.	人手が不足している

医療・高齢者領域での具体例
	•	入院時説明が理解されない
	•	ナースコールに質問が集中
	•	日常状態が記録されない
	•	スマホ操作ができない

👉 現状：
	•	人に依存
	•	デジタルは補助的

👉 提案：
	•	身体AIを情報インフラにする

⸻

③ ローカルLLM × 身体エージェント

4. 技術アーキテクチャ（実装レベル）

全体構成

[ユーザ]
  ↓ 音声
[ぬいぐるみデバイス]
  ↓ WiFi/Bluetooth
[ローカルサーバ]
  - ASR（Whisper等）
  - LLM（Llama系 / OSS）
  - RAG
  - Vector DB
  - TTS

￼

⸻

コア技術要素

1. Personal RAG
	•	個人情報・履歴・会話を蓄積

	•	関係性適応

3. 双方向ログ生成
	•	会話 → 看護ログ

4. Embodied Feedback
